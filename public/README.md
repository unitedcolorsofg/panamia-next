# Public Directory

Static assets served directly by Next.js.

## Structure

```
public/
├── img/             # Static images and photos
├── logos/           # Brand logos
├── favicon.ico      # Site favicon
├── favicon.png      # PNG favicon
├── meta_image.jpg   # Default Open Graph image
├── robots.txt       # Search engine directives
├── sitemap.xml      # Auto-generated sitemap index
└── sitemap-0.xml    # Sitemap entries
```

## Usage

Files in this directory are served from the root URL:

- `public/img/photo.png` → `https://panamia.club/img/photo.png`

In code:

```tsx
<Image src="/img/photo.png" alt="Photo" />
```

## Asset Categories

### img

Static images that don't need to be processed:

- Background patterns
- Decorative elements
- Placeholder images

### logos

Brand logos in various formats and sizes.

## SEO Files

- `robots.txt` - Controls search engine crawling
- `sitemap.xml` - Auto-generated by Next.js sitemap plugin

## Notes

- Files here are NOT processed by webpack/Next.js
- Use `next/image` for optimized image loading
- Large files increase initial bundle size
- User-uploaded content goes to Blob storage, not here
- Cloudflare Pages enforces a 25 MiB per-file limit on static assets
